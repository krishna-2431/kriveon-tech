KriveonTech UI/UX Design Specification Document
Document Owner: Principal Product Designer / Design Systems Architect
Project: KriveonTech Official Website (Phase 1)
Target Audience: UI/UX Designers, Frontend Engineers, QA Engineers
1. Design Vision
Overall Philosophy
The design philosophy for KriveonTech centers on absolute precision, mimicking the rigorous engineering standards of the code we write. The aesthetic must immediately position KriveonTech alongside premium, modern tech brands (e.g., Stripe, Vercel, Linear). 
Brand Personality
Professional: Objective and expert. 
Reassuring: Transparent processes that alleviate client anxiety. 
Technical: Confident in deep engineering capabilities, from IoT to AI/ML. 
User Emotions
Visitors should feel a sense of relief and trust. They should feel they have discovered a high-end boutique agency rather than a commodity project center. 
Design Principles
Show, Don't Tell: Utilize architecture diagrams and component trees to illustrate clean code. 
Zero Friction: Users must never have to search for a way to contact the company. 
Predictability: Layouts should follow established SaaS patterns so users intuitively know where to find information. 
Visual Goals
To transition the brand perception from an academic project center to a professional software engineering agency through a premium dark-mode UI. 
2. User Experience Goals
Visitor Journey
Discovery: Clean landing via SEO, immediately greeted by premium aesthetics. 
Validation: Value proposition read against a backdrop of professional, dark-theme UI. 
Education: Clear understanding of the 8-step process and deliverables (Code, GitHub, KT, Docs). 
Conversion: Instant connection to an engineer via persistent "Chat on WhatsApp" or "Call Now" buttons. 
Trust Building
Explicitly list the 12 deliverables (Complete Source Code, DB Files, API Docs) to differentiate from competitors. 
Prominently feature the "Knowledge Transfer Session" to alleviate post-delivery anxiety. 
Information Hierarchy
Flat and accessible hierarchy designed to prevent users from getting lost. 
All critical paths must lead to direct contact actions. 
Decision Making & Call-to-Action Strategy
Eschew traditional lead-capture forms (no POST requests for leads). 
Drive inbound inquiries exclusively through frictionless, direct Phone and WhatsApp channels. 
Accessibility
Strict adherence to WCAG 2.1 AA standards. 
3. Design Language
Minimalism & Spacing
Generous padding (standard padding: 2rem) to allow content to breathe and feel luxurious. 
Strict adherence to a 4-point grid system (standard in Tailwind CSS). 
Typography
A pairing of a premium geometric sans-serif for UI elements and a monospace font for technical accents. 
Consistency & Hierarchy
Utilize clear structural tags (<h1>, <h2>, <article>, <section>) for both visual hierarchy and semantic HTML requirements. 
Motion & Interaction
Smooth, declarative React animations using Framer Motion (essential for premium glassmorphic UI feels). 
Visual Rhythm
Glassmorphism: Subtle translucent backgrounds with background-blur (backdrop-blur-md, bg-white/5). 
Borders: 1px subtle borders on cards (border-white/10). 
Corners: Rounded UI elements with border-radius: 8px to 12px (Tailwind rounded-xl / 12px).

Font Families
Primary (UI/Readability): Inter or Geist (Premium geometric sans-serif). 
Secondary (Technical/Code): JetBrains Mono (Monospace). 
Scale & Weights

H2 (Section): 3rem (Desktop), 2rem (Mobile) | SemiBold (600) | Tracking: -0.01em

6. Spacing System
Based on the Tailwind CSS 4pt / 8pt grid system. 
2px / 4px: Micro-adjustments, borders.
8px (space-2): Inner component spacing (icon to text).
16px (space-4): Standard element spacing.
32px (space-8 / padding: 2rem): Standard container padding. 
64px (space-16): Section vertical rhythm (Mobile).
128px (space-32): Section vertical rhythm (Desktop).
7. Layout System
Breakpoints
Mobile: < 768px (Single column grid). 
Tablet: 768px - 1024px (2-column Bento boxes). 
Desktop: > 1024px (3+ column grids, max container width 1280px). 
Container Alignment
Pages utilize a centered container layout with max-width boundaries to ensure readability on ultra-wide monitors.
8. Iconography
Library: Lucide Icons. 
Style: Minimalist, geometric, consistent stroke.
Stroke Width: 1.5px to 2px.
Sizes: 16px (inline/buttons), 24px (standard), 32px (card headers), 48px (hero/timeline nodes).
Usage: Clear, icon-driven lists for deep dives into services. 
9. Imagery
Illustration Style: Abstract 3D glassmorphic representations (e.g., code compiling into a hardware chip). 
Graphics: Architecture diagrams and component trees to illustrate "Clean Code". 
Asset Optimization: Optimized WebP images loading conditionally based on viewport size. 
Tech Logos: Recognizable grid of logos for React, Node, Python, TensorFlow, ESP32, Raspberry Pi, AWS. 
10. Component Library
Primary Button
Purpose: Main CTA ("Call Now").
States: Default (Electric Blue background, white text), Hover (Glow), Active. 
Focus: Visible focus ring (ring-2 ring-blue-500 ring-offset-2 ring-offset-background). 
Secondary Button
Purpose: Secondary actions ("Chat on WhatsApp" inline).
States: Default (Transparent background, Electric Blue border), Hover (Solid), Active. 
WhatsApp FAB
Purpose: Persistent bottom-right floating action button for instant contact. 
States: Default (Floating icon), Hover (Expand with text). 
Accessibility: Must include explicit ARIA labels. 
Interaction: Utilizes the wa.me API with pre-populated message templates. 
Bento Card
Purpose: Services and features overview. 
Style: Glassmorphic (backdrop-blur-md, bg-white/5), 1px subtle borders, rounded 12px. 
States: Default, Hover (Border highlight/glow). 
Timeline Node
Purpose: Visual indicator for the 8-step process. 
States: Inactive, Active (Blue pulse). 
Navbar (Sticky)
Layout (Desktop): Logo (Left) | Home, Services, Process, FAQ (Center) | Call Now, WhatsApp (Right - Highlighted). 
Layout (Mobile): Collapses into a clean hamburger menu. Requires ARIA label. 
Footer
Layout: Logo, Tagline | Quick Links | Legal (Privacy, Terms) | Contact Info (Phone numbers). 
11. Motion Design
Engine: Framer Motion. 
Entrance Animations: Subtle fade-up (y: 20, opacity: 0 to 1) on scroll entry.
Hover Animations:
Primary Button: Drop shadow glow expansion.
Bento Cards: Subtle border color shift and 2px upward translation. 
WhatsApp FAB: Expands horizontally to reveal "Chat on WhatsApp" text. 
Timeline Pulse: Infinite subtle scale/opacity pulsing on the active Timeline Node. 
Accessibility: Respect prefers-reduced-motion CSS media queries. Disable translation/scale animations if true.

15. Accessibility
Standards: WCAG 2.1 AA compliant. 
Contrast: Electric Blue (#007AFF) on dark backgrounds must pass text contrast ratios. 
Keyboard Navigation: All CTAs and links must be fully focusable with visible focus rings (ring-2 ring-blue-500). 
ARIA Labels: Required on all icon-only buttons (WhatsApp FAB, Mobile Menu). 
Semantic HTML: Strict use of <main>, <article>, <section>, and hierarchical headers to aid screen readers. 
16. UX Writing Guidelines
Tone: Objective, expert, reassuring, and precise. 
Vocabulary: Use proper engineering terminology (e.g., "Source Code," "Architecture Diagram," "Knowledge Transfer," "IoT") to build immediate credibility. 
Expectation Management: Explicitly state exclusions (e.g., "Project Report is NOT included") to perfectly manage user expectations. 
CTA Wording: Keep it direct and action-oriented ("Call Now", "Chat on WhatsApp").

18. Interaction Design
Hover: Glassmorphic elements receive a subtle border highlight; Primary buttons receive an Electric Blue glow. 
Focus: Keyboard focus outlines (ring-2 ring-blue-500 ring-offset-2 ring-offset-background) triggered on :focus-visible. 
Transitions: All color, border, and shadow changes utilize a 150ms ease-in-out transition.
19. Future Design System (Phase 2 Roadmap)
The design system is constructed purely with decoupled components (Tailwind + shadcn/ui + Next.js App Router). This enables seamless scaling for Phase 2 without major refactoring, supporting: 
Client Portal: Secure login area to track the 8-step project lifecycle in real-time. 
Admin Dashboard: Internal management for inquiries and project statuses. 
Portfolio / Case Studies: Detailed architectural breakdowns of non-NDA projects. 
Payment Gateway: Integration for initial milestone payments securely.
